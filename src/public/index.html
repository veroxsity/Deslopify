6 tools your AI should be calling.

Every tool returns structured JSON the AI can act on. No prose, no guessing.

Before writing code
get_guidance

Returns language-specific idioms, pitfalls, and architecture notes. The AI reads this before generating a single line.

After writing code
review_code

Scores code 0-100 across robustness, idiom, generality, architecture, testability, and performance. Lists every issue with fix suggestions.

Before fixing a bug
validate_approach

Detects symptom-only fixes. If the AI is patching a leaf when the root cause is in a caller, Deslopify blocks it.

After writing code
suggest_tests

Generates structured test cases with framework-specific skeletons. Covers edge cases, error paths, and concurrency.

When adding packages
check_dependencies

Flags abandoned, deprecated, and overkill packages. Suggests lighter alternatives when a 50MB library is imported for one function.

When using unfamiliar APIs
check_api_exists

The anti-hallucination tool. Verifies whether an API actually exists in the target language and version before the AI uses it.

class ResourceManager {
    std::unique_ptr<Widget> widget;
    std::unique_ptr<Connection> conn;
public:
    ResourceManager()
        : widget{std::make_unique<Widget>()}
        , conn{std::make_unique<Connection>(...)}
    {}
    ResourceManager(const ResourceManager&) = delete;
    ResourceManager(ResourceManager&&) = default;
};

void incrementCounter() {
    const std::scoped_lock lock{mtx};
    sharedCounter++;
    doSomethingRisky();
}

void processFile(std::string_view path) {
    try {
        auto data = readFile(path);
        parse(data);
    } catch (const std::exception& e) {
        std::cerr << "Failed: " << e.what();
        throw;
    }
}
